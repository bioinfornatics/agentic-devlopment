# UX Research Evaluation

**When to load:** user journey validation, information architecture, interaction states, design taste — the strategic UX dimensions.

## Dimensions 1–4

### 1. User Intent
- What job is the user trying to complete?
- Is the primary action reachable within 2 interactions?
- Does the headline match the user goal?
- Evidence: quote the exact user-facing copy that guides or fails the user.

### 2. Information Architecture
- Clear hierarchy: heading levels, section order, label clarity.
- Navigation: can users find what they need without guessing?
- Labelling: are CTAs and form fields named for what they do?
- Evidence: quote the labels; note the visual hierarchy order.

### 3. Interaction States
- Does the UI have: loading, empty, error, success, disabled, focus, offline states?
- Are error messages actionable ("Enter a valid email" not "Invalid input")?
- Evidence: trigger each state and screenshot or describe observed behaviour.

### 4. Visual Taste
- Deliberate spacing, contrast, rhythm, typography — not generic AI gradients.
- Consistent spacing scale? Type hierarchy clear?
- Evidence: cite specific violations with element + expected vs. observed.

## Self-validation (UX)
- [ ] User intent finding cites exact user-facing copy
- [ ] IA findings cite specific labels or navigation paths observed
- [ ] Every state finding was triggered (not assumed from code)
- [ ] Design taste findings reference design conventions, not preferences
