# Responsibility matrix

| Agent | Purpose / responsibility evidence | Loaded skills | Invoked by | Review / authority note |
|---|---|---|---|---|
| `architect` | Use PROACTIVELY when planning a new feature, making a technology choice, or touching a system boundary. Produces ADRs, design proposals, and trade-off analyses. Read-only. | agentic-devlopment, design-systems-arch, sdd | recipes/evals per static inventory | Independent unless explicitly reviewer/escalation |
| `codebase-researcher` | Read-only codebase researcher. Use to map architecture, trace blast radius, and gather evidence before planning or implementation. Safe to run in parallel. Produces file-level evidence reports. | agentic-devlopment, goose-orchestration | recipes/evals per static inventory | Independent unless explicitly reviewer/escalation |
| `harness-judge` | Evidence-first LLM-as-judge for the Goose agentic development harness. Audits completed sessions and harness artifacts across prompt, context, loop, skills, agents, recipes, frameworks, orchestration, ontology/knowledge-graph integrity, and layer-delta value. Invoke for read-only grading and pre-merge harness audits; do not invoke for implementation. | harness-judge | recipes/evals per static inventory | Independent unless explicitly reviewer/escalation |
| `implementation-worker` | Implementation specialist for scoped Beads issues. Invoke when a bead is claimed and ready for coding. Enforces TDD (tests first), minimal blast radius, and complete handoff. | agentic-devlopment, beads, knowledge-graph, s, sdd | recipes/evals per static inventory | Independent unless explicitly reviewer/escalation |
| `orchestrator` | Lead orchestrator for the SDD+TDD loop. Calls load() to discover available specialists and routes to the best match by description. Owns scope partitioning, delegation, and synthesis across all SDD phases. Invoke for work spanning two or more specialist agents or phases. | agentic-devlopment, beads, goose-orchestration | recipes/evals per static inventory | Independent unless explicitly reviewer/escalation |
| `planner` | Beads dependency graph specialist. Invoked BY product-owner or architect when a product goal requires ordered Beads with dependency chains across more than 5 interdependent issues or multiple files, agents, and sessions. | beads | recipes/evals per static inventory | Independent unless explicitly reviewer/escalation |
| `principal-engineer` | Use when a change touches shared infrastructure, public APIs, breaking changes, or architectural boundaries. Escalation path after 2+ BLOCK verdicts from review-critic. Read-only. | agentic-devlopment, code-review | recipes/evals per static inventory | Independent unless explicitly reviewer/escalation |
| `product-owner` | Product Owner — owns the full backlog lifecycle: user story definition, PRD quality gate (≥85/100), Beads epic/story creation, priority and assignee management. Invoke at the start of any new feature, for backlog refinement, or to assign work to specialist agents. | agentic-devlopment, agentic-ux, s, sdd | recipes/evals per static inventory | Independent unless explicitly reviewer/escalation |
| `qa-automation` | QA automation engineer. Invoked after implementation to design and execute the automated test strategy: unit, integration, E2E. Manages flaky test quarantine, CI integration, and coverage reporting. | agentic-devlopment, systematic-debugging, webapp-testing | recipes/evals per static inventory | Independent unless explicitly reviewer/escalation |
| `review-critic` | Critical code and Beads handoff reviewer. Invoke after any implementation, before closing a bead or merging. Returns severity-ranked findings with proof and a verdict: APPROVE, PASS-WITH-NITS, or BLOCK. | code-review, knowledge-graph, s | recipes/evals per static inventory | Independent unless explicitly reviewer/escalation |
| `tdd-guide` | Use PROACTIVELY before any new feature implementation or bug fix. Enforces write-tests-first with 80%+ coverage gates. Invoke before implementation-worker to establish the failing RED test first. | agentic-devlopment, sdd, systematic-debugging | recipes/evals per static inventory | Independent unless explicitly reviewer/escalation |
| `ui-designer` | User interface designer. Invoke after UX research validates the direction: for component design, design system consistency, WCAG 2.2 AA compliance, and visual implementation review. | atomic-design, cognitive-ux, design-systems-arch, frontend-blueprint, ui-quality, ux-quality | recipes/evals per static inventory | Independent unless explicitly reviewer/escalation |
| `ux-researcher` | User experience researcher. Invoke when understanding user needs, defining personas, mapping user journeys, planning usability tests, or validating product decisions with user evidence. | agentic-ux, cognitive-ux, ux-quality | recipes/evals per static inventory | Independent unless explicitly reviewer/escalation |

## Coverage notes

All 13 agents have Required Skill Load guards per `scripts/check-consistency.py` output. Responsibility overlap remains between `review-critic`, `principal-engineer`, and `harness-judge` and is intentional only when challenge/judge independence is preserved.
